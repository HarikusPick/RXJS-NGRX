shareReplay(1) — HTTP çağrısını tekrarlama

    

      Aynı Observable'a birden fazla abone olunduğunda her biri ayrı HTTP isteği tetikler.
      shareReplay(1) ile tek istek yapılır, sonuç cache'lenir.
    

    

    

      
      
    

    

      @if (!shareReplayLogs().length) {
        Bir buton seç...
      }
      @for (line of shareReplayLogs(); track $index) {
        
{{ line }}

      }
    

    
💡 Tarayıcı Network sekmesini aç — shareReplay YOK durumunda 2 ayrı istek görürsün.

  


  
  

    
exhaustMap — Meşgulken yeni isteği yoksay

    

      Form submit gibi durumlarda: kullanıcı hızlı tıklarsa sadece ilk istek çalışır,
      istek tamamlanana kadar yenileri sessizce yoksayılır.
    

    

    

      
    

    

      @if (!exhaustLogs().length) {
        Butona birkaç kez hızlıca tıkla...
      }
      @for (line of exhaustLogs(); track $index) {
        
{{ line }}

      }
    

    
💡 ⚡ satırları yoksayılan tıklamaları gösterir. switchMap kullansaydık önceki iptal edilirdi.

  


  
  

    
retry() — Hata sonrası otomatik tekrar

    

      retry(n) ile N kez tekrar dener. defer() ile birlikte
      her deneme için taze bir Observable üretilir.
    

    

    
    

      @if (!retryLogs().length) {
        Butona tıkla...
      }
      @for (line of retryLogs(); track $index) {
        
{{ line }}

      }
    

    
💡 İlk 2 deneme simüle hata fırlatır, 3. deneme gerçek HTTP isteği yapar.

  


  
  

    
takeUntilDestroyed() — Memory leak önleme

    

      subscribe() ile açılan abonelikler kapatılmazsa bileşen destroy
      olsa da çalışmaya devam eder. Angular 16+ ile gelen modern çözüm.
    

    

    

      Memory Leak Neden Olur?
      
    
        
  • interval(), fromEvent() gibi sonsuz Observable'lar iptal edilmezse çalışmaya devam eder
    • 
        
  • Bileşen yok edilse bile callback'ler çalışır → null pointer hatası, gereksiz hesaplama
    • 
        
  • takeUntilDestroyed, DestroyRef ile entegre — bileşen ömrüne bağlı
    •

withLatestFrom — Tetiklemeden değer al

    

      combineLatest her iki kaynak değişince tetikler.
      withLatestFrom yalnızca sol kaynak (trigger) değişince tetikler,
      sağ kaynaktan sadece anlık değer alır.
    

    

    

      

        Otomatik sayaç
        {{ counterSignal() }}
      

      
    

    

      @if (!captureLog().length) {
        Butona bas — sayacın anlık değerini yakala...
      }
      @for (line of captureLog(); track $index) {
        
{{ line }}

      }
    

    
💡 Sayaç her saniye artar ama butona basmadan hiçbir şey tetiklenmez. Butona basınca anlık değer yakalanır.

  


  
  

    
defer() — Lazy Observable fabrikası

    

      Her subscribe'da fabrika fonksiyonu yeniden çalışır. Retry ile mükemmel uyum sağlar.
      Her deneme taze bir Observable üretir.
    

    

    
    

      @if (!deferLogs().length) {
        Butona tıkla...
      }
      @for (line of deferLogs(); track $index) {
        
{{ line }}

      }
    

    
💡 İki abone farklı zaman damgaları alır çünkü defer(), her subscribe'da fabrikayı yeniden çalıştırır.
