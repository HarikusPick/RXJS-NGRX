      Ne zaman ComponentStore, ne zaman global Store?
      

        

          Global Store
          
    
            
  • Birden fazla bileşende paylaşılan veri
    • 
            
  • Route değişimlerinde korunması gereken state
    • 
            
  • DevTools ile debug edilmesi gereken veri
    • 
            
  • Kullanıcı oturumu, sepet, bildirimler
    • 
          

        

        

          ComponentStore
          
    
            
  • Yalnızca bir bileşene özgü state
    • 
            
  • Modal, dropdown, wizard adımları
    • 
            
  • Sayfa içi geçici durum
    • 
            
  • Bileşen kapandığında temizlenmeli
    • 
          

        

      

    

  


  
  

    
İleri Effect Kalıpları

    

      concatLatestFrom — effects içinde güvenli selector okuma.
      Birden fazla action dispatch etme.
    

    

    

      concatLatestFrom vs withLatestFrom
      
    
        
  • withLatestFrom: selector subscription anında store'un son değerini alır — timing race condition riski var
    • 
        
  • concatLatestFrom: NgRx'in kendi operatörü, action ile selector'ı senkronize eder — daha güvenli
    • 
        
  • Efekt içinde store'dan değer okuyacaksan concatLatestFrom tercih et
    •
